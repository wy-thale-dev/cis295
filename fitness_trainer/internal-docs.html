Container Queries

Our website has moved to implement Container Queries as part of modernizing our design. Currently, articles on the main page are querying their parent to resize their images depending on the width of the parent container. This allows changes to happen on dynamic elements like grids without relying on basing their size on the user's viewport width.

Querying Container Width

By querying the width of a parent container, we can easily set breakpoints for individual UI elements that are independent of the width of the tab. This way, we can make dynamic elements that scale well across all screen widths. We can also improve the flexibility of dynamic content.
This horizontally-resizable demo shows how the format of a grid can change while the webpage remains static. Displayed container widths will disappear as you shrink past them.

HTML

                    
<article>
    <p>900px</p>
    <p>750px</p>
    <p>600px</p>
    <p>450px</p>
    <p>300px</p>
    <p>150px</p>
</article>

CSS


@container (max-width:900px) {
    article {
        p:nth-child(1) {
            color: $tree-darker;
        }
    }
}
@container (max-width:750px) {
    article {
        p:nth-child(2) {
            color: $tree-darker;
        }
    }
}
@container (max-width:600px) {
    article {
        grid-template-columns: repeat(2, 1fr);
        p:nth-child(3) {
            color: $tree-darker;
        }
    }
}
@container (max-width:450px) {
    article {
        p:nth-child(4) {
            color: $tree-darker;
        }
    }
}
@container (max-width:300px) {
    article {
        grid-template-columns: repeat(1, 1fr);
        p:nth-child(5) {
            color: $tree-darker;
        }
    }
}
@container (max-width:150px) {
    article {
        p:nth-child(6) {
            color: $tree-darker;
        }
    }
}

This feature should work with modern versions of most browsers. That being said, remember that not every situation requires these queries. Simple max-widths and percentage based units can keep layouts such as this website's scalable without creating confusing or cluttered code.

To read more about these tools, look through this article by Gabriel Shoyombo on Smashing Magazine's website. It provides a brief overview of container queries and links to several resources that go further in depth on the topic.
